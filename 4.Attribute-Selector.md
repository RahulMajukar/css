# 🎯 CSS Attribute Selectors - Precision Targeting

**Attribute Selectors** let you target elements by their **attributes** and **values**.

**Why use them?**
> No need for extra classes! Target by `type`, `href`, `data-*`, etc.

## Basic Syntax

```
[attribute]          → Has the attribute
[attribute="value"]  → Exact match
[attribute~="value"] → Contains word
[attribute|="value"] → Starts with value (hyphen separated)
[attribute^="value"] → Starts with
[attribute$="value"] → Ends with
[attribute*="value"] → Contains anywhere
```

---

## 🔑 6 Attribute Selector Types (With Examples)

### 1. `[attribute]` - Has Attribute (Any Value)

**Targets ANY element with that attribute:**

```css
/* Any button with a data-action */
[data-action] {
  cursor: pointer;
  padding: 8px 16px;
}

/* Any input with required */
input[required] {
  border-left: 4px solid #007bff;
}
```

**HTML:**
```html
<button data-action="save">Save</button>  <!-- Styled! -->
<input required>                        <!-- Red border! -->
```

### 2. `[attribute="value"]` - Exact Match ⭐

**Exact value match:**

```css
/* Text inputs */
input[type="text"] {
  padding: 8px 12px;
  border: 1px solid #ddd;
}

/* Submit buttons */
input[type="submit"] {
  background: #007bff;
  color: white;
  border: none;
  padding: 10px 20px;
  border-radius: 4px;
}
```

### 3. `[attribute^="value"]` - Starts With

**Perfect for protocols, classes:**

```css
/* External HTTPS links */
a[href^="https"] {
  color: #007bff;
  text-decoration: none;
}

/* Primary buttons */
.btn[data-primary] {
  background: #007bff;
  color: white;
}

/* PDF downloads */
a[href$=".pdf"] {
  color: red;
}
```

### 4. `[attribute$="value"]` - Ends With

```css
/* Image links */
a[href$=".jpg"],
a[href$=".png"],
a[href$=".webp"] {
  border: 2px solid #eee;
}

/* Required fields */
input[required],
select[required],
textarea[required] {
  background: #fff8f0;
}
```

### 5. `[attribute*="value"]` - Contains Anywhere

```css
/* Any element with "modal" in data-role */
[data-role*="modal"] {
  position: fixed;
  background: white;
}

/* Buttons with "loading" */
.btn[data-state*="loading"] {
  opacity: 0.7;
  cursor: not-allowed;
}
```

### 6. `[attribute~="value"]` - Word List (Space Separated)

```css
/* class="btn primary large" */
.btn[class~="primary"] {
  background: #007bff;
}
```

## 🎨 Real-World Examples

### 1. **Perfect Form Styling** (No Classes Needed!)

```css
/* All text inputs */
input[type="text"],
input[type="email"],
input[type="password"],
input[type="url"],
input[type="tel"] {
  padding: 12px 16px;
  border: 2px solid #e1e5e9;
  border-radius: 8px;
  font-size: 16px;
}

/* Textarea */
textarea {
  resize: vertical;
  min-height: 120px;
}

/* Submit/Primary buttons */
input[type="submit"],
button[type="submit"] {
  background: #10b981;
  color: white;
  border: none;
  padding: 14px 28px;
  border-radius: 8px;
  font-weight: 600;
  cursor: pointer;
}

/* Disabled state */
input:disabled,
button:disabled,
select:disabled {
  opacity: 0.5;
  cursor: not-allowed;
  background: #f3f4f6;
}
```

### 2. **Smart Link Styling**

```css
/* External links */
a[href^="http"]:not([href*="//yourdomain.com"]) {
  color: #007bff;
}

/* Email links */
a[href^="mailto:"] {
  color: #d63384;
}

/* Phone links */
a[href^="tel:"] {
  color: #10b981;
}
```

### 3. **Data Attributes System** ⭐ Modern Pattern

```css
/* Any element with data-size */
[data-size="small"] {
  font-size: 0.875rem;
  padding: 0.5rem;
}

[data-size="medium"] {
  font-size: 1rem;
  padding: 1rem;
}

[data-size="large"] {
  font-size: 1.25rem;
  padding: 1.5rem;
}

/* Status badges */
[data-status="success"] {
  background: #d1fae5;
  color: #065f46;
}

[data-status="warning"] {
  background: #fef3c7;
  color: #92400e;
}

[data-status="error"] {
  background: #fee2e2;
  color: #991b1b;
}
```

**HTML:**
```html
<button data-action="save" data-size="medium" data-status="success">
  Save Changes
</button>
```

### 4. **Interactive Components**

```css
/* Loading states */
[data-loading="true"] {
  opacity: 0.6;
  pointer-events: none;
}

/* Drag & drop */
[data-draggable="true"] {
  cursor: grab;
}

[data-draggable="true"]:active {
  cursor: grabbing;
}
```

## 📱 Responsive Attribute Selectors

```css
/* Mobile-specific */
@media (max-width: 768px) {
  input[type="text"][data-full-width] {
    width: 100%;
  }
}

## 🎯 Complete Attribute Selector Reference

| Selector | Meaning | Example | Matches |
|----------|---------|---------|---------|
| `[title]` | Has title | `img[title]` | `<img title="hi">` |
| `[type="text"]` | Exact | `input[type="text"]` | `<input type="text">` |
| `[href^="https"]` | Starts | `a[href^="https"]` | `https://`, `https://sub.` |
| `[href$=".pdf"]` | Ends | `a[href$=".pdf"]` | `doc.pdf`, `file.doc.pdf` |
| `[class*="btn"]` | Contains | `.item[class*="btn"]` | `class="big-btn primary"` |
| `[data-active]` | Has attr | `[data-active]` | `<div data-active>` |

    <input type="submit" value="Disabled" disabled>
  </form>

</html>

## 🚀 Pro Tips

1. **Combine with pseudo-classes:**
```css
input[type="text"]:focus {
  border-color: var(--primary-color);
}
```

2. **Use for ARIA states:**
```css
[aria-expanded="true"] {
  background: #fef3c7;
}
```

3. **Perfect for third-party components** (no class control)
