# 🎨 CSS Display Property - Complete Guide

**display** controls how an element behaves in the layout flow.

**In simple words:**  
It decides:
> 👉 Does the element start on a new line?  
> 👉 Can it have `width`/`height`?  
> 👉 Can other elements sit next to it?

## Display Property Table

| display value | New line? | Width/Height? | Sits inline? | Typical use |
|---------------|-----------|---------------|--------------|-------------|
| `block` | ✅ Yes | ✅ Yes | ❌ No | sections, cards, containers |
| `inline` | ❌ No | ❌ No | ✅ Yes | text, links |
| `inline-block` | ❌ No | ✅ Yes | ✅ Yes | buttons, badges |
| `none` | ❌ No element | ❌ No element | ❌ No element | hide UI |
| `table` | ✅ (table-like) | ✅ Yes | ❌ No | legacy layouts |

---

## ✅ 1. `display: block`

**👉 Meaning**  
A **block element**:
- Always starts on a **new line**
- Takes **full available width** by default
- Respects: `width`, `height`, `margin`, `padding`

**Default block elements:**
```html
div, p, h1-h6, section, article, form
```

**Example:**
```css
.card {
  display: block;
  width: 300px;
  height: 120px;
  background: lightblue;
  margin: 20px;
  padding: 20px;
}
```

**Behavior:**
```
[ block element       ]
[ next block goes below ]
```

---

## ✅ 2. `display: inline`

**👉 Meaning**  
An **inline element**:
- Does **NOT** start on a new line
- Only takes as much **width as its content**
- **Ignores** `width` and `height`

**Default inline elements:**
```html
span, a, strong, em
```

**Example:**
```css
span {
  display: inline;
  width: 200px;   /* ❌ IGNORED */
  height: 50px;   /* ❌ IGNORED */
  background: yellow;
  padding: 10px;
}
```

**Behavior:**
```
text [inline] text [inline] text
All stay in the same line!
```

**🚨 Important limitations:**
- ❌ Cannot control box size (`width`/`height`)
- ❌ Vertical `margin` has almost no effect

**When to use inline:**
- Text styling
- Links inside paragraphs
- Icons inside text

**Key points:**
> ✔ Stays in the same line  
> ✔ `width`/`height` do NOT work  
> ✔ Perfect for text-level elements

---

## ✅ 3. `display: inline-block` ⭐ **MUST-KNOW**

**👉 Meaning**  
`inline-block` is a **mix** of:
- `inline` → sits next to other elements
- `block` → allows `width` & `height`

**Example:**
```css
.badge {
  display: inline-block;
  width: 80px;
  height: 30px;
  background: #28a745;
  color: white;
  text-align: center;
  margin-right: 10px;
}
```

**HTML:**
```html
<span class="badge">New</span>
<span class="badge">Hot</span>
<span class="badge">Sale</span>
```

**Behavior:**
```
[inline-block] [inline-block] [inline-block]
But each one has its own box + can be sized!
```

**Real project use cases:**
```css
/* Buttons in toolbar */
.toolbar button {
  display: inline-block;
  margin-right: 10px;
}

/* Tags system */
.tag {
  display: inline-block;
  padding: 4px 12px;
  background: #eee;
  border-radius: 20px;
}
```

**Key points:**
> ✔ Sits in one line  
> ✔ `width`/`height` **WORK**  
> ✔ `margin` & `padding` **WORK**

---

## ✅ 4. `display: none`

**👉 Meaning**  
The element is **completely removed** from layout.

**Example:**
```css
.modal {
  display: none;  /* Hidden */
}

.modal.active {
  display: block; /* Show */
}
```

**What happens?**
```
✅ Element is NOT visible
✅ Element takes NO space
✅ Browser acts like it doesn't exist
```

**VS `visibility: hidden`:**
```css
display: none;
👉 Removed from layout completely

visibility: hidden;
👉 Invisible BUT space is kept
```

---

## ✅ 5. `display: table` (Legacy)

**👉 Meaning**  
Makes elements behave like HTML **tables**—without `<table>` tags.

**Example:**
```css
.wrapper {
  display: table;
  width: 100%;
}

.row {
  display: table-row;
}

.cell {
  display: table-cell;
  padding: 20px;
  border: 1px solid #ddd;
}
```

**HTML:**
```html
<div class="wrapper">
  <div class="row">
    <div class="cell">Name</div>
    <div class="cell">Price</div>
    <div class="cell">Status</div>
  </div>
</div>
```

**Why does this exist?**  
Before Flexbox/Grid, used for:
- Equal height columns
- Vertical centering
- Complex table-like layouts

**Modern replacement:** Use Flexbox or Grid instead!
